### Simple notes about our tools/ expectations.

1. some tools require third-party trusted pip-installed packages, so check the requirements.txt file
2. our goal is to create efficient tools which are self-contained in one file; in the process of writing clean code this is
often unpleasant for maintainance, as several tools will reuse the same underlying code over and over again. to differential tools
that require other self-contained files, we call these "hools" as in "heavy" tools. They are quite lean and a work in progress.
3. tools change and packages break, but we aim to make these as stable as possible

any problems you know how to reach me.

don't over think it.
